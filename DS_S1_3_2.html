Unidad 1 Introducción

1.3 Modelación

  • Método general de modelación
    • Modelo conceptual
      Un modelo conceptual de modelación es una representación simplificada y abstracta de un sistema o fenómeno que se utiliza para comprender su estructura, relaciones y comportamiento. A diferencia de otros tipos de modelos, como los modelos matemáticos o computacionales, un modelo conceptual se enfoca en la comprensión conceptual y visual del sistema, sin entrar en detalles matemáticos o técnicos. El objetivo principal de un modelo conceptual de modelación es capturar las ideas clave, los componentes principales y las relaciones fundamentales del sistema, de manera que se pueda obtener una comprensión clara y compartida entre las partes interesadas. A menudo, estos modelos se crean en las etapas iniciales del proceso de modelado, ayudando a definir y comunicar el alcance y los requisitos del modelo.
      Un modelo conceptual puede ser expresado de diferentes formas, entre las cuales se encuentran la descripción verbal, los diagramas de causa a efecto y los modelos gráficos. A continuación, se detalla cada una de estas representaciones:
      • Descripción verbal:
        Consiste en utilizar palabras y lenguaje natural para explicar y describir los componentes y las relaciones de un sistema o fenómeno. En este enfoque, se utilizan oraciones, párrafos o documentos para transmitir la información y las ideas clave del modelo. La descripción verbal puede incluir definiciones, explicaciones de conceptos, secuencias de eventos y relaciones causales, entre otros detalles relevantes.
      • Diagramas de causa a efecto (diagramas de espina de pescado o diagramas de Ishikawa):
        Estos diagramas son herramientas visuales que ayudan a identificar y mostrar las relaciones causales entre diferentes factores o variables que contribuyen a un efecto o resultado específico. En un diagrama de causa a efecto, también conocido como diagrama de espina de pescado debido a su forma, se utiliza una línea central (la "columna vertebral") para representar el efecto o resultado deseado, y se ramifican líneas secundarias que representan diferentes categorías de factores o causas que pueden contribuir a ese efecto. Estas categorías de causas se dividen en subcategorías más específicas, y así sucesivamente, hasta que se identifican los factores individuales relevantes.
      • Modelo gráfico:
        Los modelos gráficos son representaciones visuales que utilizan símbolos, iconos, diagramas o gráficos para mostrar la estructura, las relaciones y las interacciones de un sistema. Pueden incluir diferentes tipos de diagramas, como diagramas de flujo, diagramas de bloques, diagramas de redes, diagramas de estados, entre otros. Estos modelos gráficos permiten una representación más intuitiva y fácil de comprender del sistema, ya que utilizan elementos visuales y conexiones entre ellos para mostrar cómo se relacionan los componentes y cómo fluye la información o los eventos dentro del sistema.
      Cada una de estas representaciones puede ser utilizada de acuerdo con las necesidades y preferencias del modelador y los usuarios del modelo. A menudo, se combinan varias formas de representación para crear un modelo conceptual más completo y comprensible. El objetivo principal de todas estas representaciones es capturar y comunicar de manera clara y efectiva la estructura, las relaciones y el comportamiento del sistema o fenómeno en estudio.
    • Modelo matemático:
      En esta etapa, se traduce el modelo conceptual en un conjunto de ecuaciones matemáticas que describen las relaciones cuantitativas entre las variables del sistema. El modelo matemático utiliza símbolos, operaciones matemáticas y funciones para representar estas relaciones.
    • Programa computacional:
      En esta etapa, se implementa el modelo matemático en un programa computacional utilizando un lenguaje de programación adecuado. El programa computacional es una implementación práctica del modelo y permite realizar cálculos, simular el comportamiento del sistema y obtener resultados numéricos.
    • Interpretación de resultados:
      Una vez que se ejecuta el programa computacional y se obtienen los resultados, se lleva a cabo la interpretación de los mismos. Esto implica analizar y comprender los resultados en relación con el sistema modelado, identificar tendencias, patrones y relaciones significativas, y extraer conclusiones relevantes.
    • Evaluación del modelo:
      En esta etapa, se evalúa el desempeño del modelo en términos de su exactitud y su comportamiento. La exactitud se evalúa comparando los resultados del modelo con datos reales o mediciones del sistema. El comportamiento se evalúa en relación con los objetivos y los criterios de éxito establecidos al inicio del proceso de modelación. La evaluación del modelo proporciona información sobre la calidad y la utilidad del modelo, y puede guiar ajustes o mejoras en el mismo. Se pueden utilizar métricas de evaluación, como el error absoluto medio, el coeficiente de determinación (R^2) y el análisis de sensibilidad, para cuantificar la precisión del modelo y su capacidad para predecir el comportamiento del sistema.
      • Exactitud: modelo vs sistema real: La exactitud se refiere a qué tan cerca está el modelo de representar el sistema real. Para evaluar la exactitud del modelo, se comparan los resultados obtenidos del modelo con datos reales, mediciones o información confiable del sistema. Se pueden utilizar métricas de evaluación, como el error absoluto medio, el coeficiente de determinación (R^2), la raíz del error cuadrático medio (RMSE) u otras medidas pertinentes, para cuantificar la diferencia entre el modelo y el sistema real.
      • Resultado del comportamiento: Además de la exactitud, se evalúa el comportamiento del modelo en relación con los objetivos y los criterios de éxito definidos al inicio del proceso de modelación. Se examina si el modelo es capaz de capturar adecuadamente el comportamiento y las tendencias del sistema, si puede predecir de manera precisa los resultados deseados y si cumple con los requisitos y las restricciones establecidos. Se pueden considerar indicadores clave de rendimiento para evaluar el éxito del modelo en términos de los resultados esperados.
      La evaluación del modelo puede implicar iteraciones y ajustes en el modelo, así como en los datos utilizados y en las suposiciones realizadas, con el fin de mejorar su exactitud y comportamiento. Es importante tener en cuenta que ningún modelo es perfecto y siempre existirá cierto nivel de incertidumbre y margen de error. La evaluación del modelo proporciona información valiosa para determinar la calidad y la utilidad del modelo, así como para identificar áreas de mejora o ajuste. Es importante destacar que la evaluación del modelo no es un proceso único, sino que puede realizarse de manera continua a medida que se adquiere más información o se obtienen nuevos datos. Además, la evaluación del modelo puede involucrar la retroalimentación de las partes interesadas y la validación externa, en la cual expertos o investigadores externos revisan y evalúan el modelo para brindar una perspectiva imparcial y objetiva.
